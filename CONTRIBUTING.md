# Contribuindo

Obrigado por contribuir com o Hair Day.

## Fluxo recomendado

1. Faça um fork do projeto.
2. Crie uma branch para sua alteração.
3. Implemente a mudança com foco e escopo claro.
4. Teste localmente com a API mock e o frontend em execução.
5. Abra um Pull Request descrevendo objetivo, contexto e impacto.

## Ambiente local

```bash
npm install
npm run server
npm run dev
```

## Boas práticas

- Mantenha os módulos pequenos e coesos.
- Evite mudanças não relacionadas no mesmo PR.
- Documente mudanças de comportamento no README quando necessário.

## Reportar problemas

Ao abrir uma issue, inclua:

- passos para reproduzir
- comportamento esperado
- comportamento atual
- logs/prints relevantes
